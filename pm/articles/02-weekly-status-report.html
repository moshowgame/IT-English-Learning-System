Weekly Status Report — Speaking It Out

Article 02 · Module 1: Daily PM Communication · PM English

⏱ 8 min read 🟢 Beginner-friendly 🎯 RAG bullets, milestone movement, RAID highlights, asks — format an MD will actually read

1. Scenario 场景背景

It is Friday morning. You are the PM for the Global Sanctions Screening Platform upgrade. Every Friday at 10:00 HKT you walk a 5-minute Weekly Status in the workstream leads' forum. On the call: Tom (Workstream Lead — Screening Engine, London), Anita (Workstream Lead — UI, Hong Kong), Vikram (Workstream Lead — Data Migration, Bangalore), and Helen (PMO, who tracks SteerCo minutes).

周五上午，你是全球制裁名单筛查平台升级的 PM。每个周五 10:00 香港时间，你在工作流负责人论坛上做一个 5 分钟的周报口头汇报。参会有：Tom（筛查引擎工作流负责人，伦敦）、Anita（UI 工作流负责人，香港）、Vikram（数据迁移工作流负责人，班加罗尔），以及 Helen（PMO，跟踪 SteerCo 会议纪要）。

The forum is your chance to synchronise the workstreams, surface cross-workstream risks, and brief Helen so the SteerCo pack is consistent. You are not the only one speaking — each workstream lead reports in 60 seconds. But the project-level summary is yours.

这个论坛是你同步各工作流、暴露跨工作流风险、让 Helen 知道 SteerCo 资料保持一致的机会。你不是唯一讲话的人——每个工作流负责人汇报 60 秒。但项目层面的总结是你的。

2. Dialogue 英文对话

Helen (PMO): Right, let's start. PM, weekly headline and RAG.好，开始。PM，一句话和 RAG。
You (PM): This week: project RAG is amber. Headline: we are on track for the OFAC milestone on 14th, but the List Management Module slipped by 3 days due to a vendor API schema change.本周：项目 RAG 琥珀色。一句话：14 号 OFAC 里程碑按计划，但名单管理模块因厂商 API schema 变更推迟 3 天。
You (PM): Last week we closed M2 design and started M3 build. This week: Screening Engine hit 78% complete, UI is at 62%, Data Migration is behind plan by 3 days — root cause is the vendor schema change. Next week: M3 build continues, M4 code freeze on Friday.上周关闭 M2 设计，启动 M3 构建。本周：筛查引擎 78% 完成，UI 62%，数据迁移落后 3 天——根因是厂商 schema 变更。下周：M3 构建继续，M4 周五 code freeze。
You (PM): RAID highlights: one new risk — the vendor hasn't confirmed the go-live cutover window. Probability medium, impact high. Mitigation: I'm hosting a cutover planning workshop with the vendor on Monday. No new issues, no escalations.RAID 要点：新登记一个风险——厂商未确认上线切换窗口。概率中，影响高。缓解：周一和厂商开切换规划工作坊。无新问题，无升级。
You (PM): Asks: Sponsor to sign-off on the cutover weekend by Wednesday. Group Cyber to confirm the penetration test report by Friday. Helen, can you reflect both asks in the SteerCo pack?Asks：Sponsor 周三前签批切换周末窗口。集团 Cyber 周五前确认渗透测试报告。Helen，能在 SteerCo 资料中反映这两个 ask 吗？
Helen: Got it, I'll add them to the decisions page.收到，我加到决策页。
You (PM): Thanks. Tom, Screening Engine detail?谢。Tom，筛查引擎细节？

3. Key Phrases 关键句型

  • Last week Last week we closed M2 design and started M3 build. 上周我们关闭 M2 设计，启动 M3 构建 The "last week / this week / next week" rhythm is universal. Senior leaders love it.
  • Movement M3 build is 78% complete, ahead of plan by 2 days. M3 构建完成 78%，比计划提前 2 天 Always pair % with a direction. Numbers without direction are noise.
  • Root cause Root cause is the vendor API schema change. 根因是厂商 API schema 变更 Always name the root cause of a slip. "We slipped" is not enough.
  • Risk One new risk logged, probability medium, impact high. 新登记一个风险，概率中，影响高 Format: count + name + P×I. Same every time.
  • Mitigation Mitigation: I'm hosting a workshop on Monday. 缓解：周一我主持一个工作坊 Every risk needs an owner and a date. "We're looking at it" is not a mitigation.
  • Ask Sponsor to sign-off by Wednesday. Sponsor 周三前签批 Asks are short: who, what, by when. Don't explain why in the headline — that's the deck.
  • PMO loop Helen, can you reflect both asks in the SteerCo pack? Helen，SteerCo 资料能反映这两个 ask 吗？ PMs always close the loop with PMO. If it's not in the pack, it didn't happen.
  • Hand-off Tom, Screening Engine detail? Tom，筛查引擎细节？ Pass the mic by name. Don't say "anyone else" or the silence will eat your meeting.

4. Vocabulary 词汇表

  • Workstream n. 工作流 A sub-track within a project, usually with its own lead, plan, and risks.
  • Code freeze n. 代码冻结 The point in the plan where no new code is accepted into the release branch.
  • Cutover n. 切换 The weekend or window when traffic moves from old system to new system.
  • Vendor schema change n. 厂商 schema 变更 When a third-party changes the structure of their API or file.
  • OFAC n. 美国财政部海外资产控制办公室 US Treasury's sanctions enforcement arm — a key regulator for sanctions screening.
  • PMO n. 项目管理办公室 Project Management Office — owns governance, SteerCo packs, and minutes.
  • SteerCo pack n. Steering Committee 资料 The deck distributed before the Steering Committee meeting.
  • Workshop n. 工作坊 A facilitated working session, often used to unblock a specific decision.
  • Penetration test n. 渗透测试 Security test where ethical hackers try to break into the system.
  • Sign-off n. 签字 / 批准 Formal approval from a named authority — usually documented in writing.
  • Decisions page n. 决策页 A standard slide in the SteerCo pack listing decisions needed this week.

5. Grammar Notes 语法点

① "Last week / This week / Next week" — the three-bullet rhythm

The "Last / This / Next" rhythm is the most senior-friendly structure in PM English. Last week we closed X. This week we are doing Y. Next week we will do Z. Anyone can read it in 15 seconds. Memorise the rhythm.

"上周 / 本周 / 下周"的三段式是 PM 英语中最高层友好的结构。任何人都能在 15 秒内读完。

② "Slipped by N days" — quantify the slip

Never say "we slipped a bit" or "we're a little behind". Always quantify: slipped by 3 days / 1 week / 1 sprint. The number is the basis for the recovery plan and the impact analysis.

永远不要说"我们有点延期"或"稍微落后"。要量化：slipped by 3 days / 1 week / 1 sprint。

③ "Probability × Impact" — the universal risk format

Every risk gets a probability and an impact, in that order. Probability low, impact high. Probability medium, impact medium. The pair is what makes the risk scoreable and comparable across the RAID log.

每个风险都给出概率和影响：Probability low, impact high. 这两个组合是 RAID log 中可评分可比对的基础。

6. Cultural Tips 文化 / 职场 Tips

  • "Last / This / Next" wins every time. It's the format senior stakeholders read in 15 seconds. If you master one PM structure, master this one.
  • Root cause is not optional. A slip without a root cause looks like incompetence. A slip with a root cause looks like control.
  • Mitigations have owners and dates. "We're looking at it" is not a mitigation. "Anita owns it, due Friday" is.
  • Asks are short. "Sponsor to sign-off the cutover weekend by Wednesday." That's it. Save the why for the deck.
  • Close the loop with PMO. "Helen, can you reflect this in the SteerCo pack?" — this is how you make sure your ask doesn't die in someone's inbox.
  • No new issues is a feature, not a bug. A clean RAID is a sign of control. Don't invent problems to look busy.
  • Quantify in PM time, not engineering time. "Behind plan by 3 days" beats "behind by 2 sprints". Senior leaders think in days and weeks.
  • Regulator milestones anchor everything. Always re-anchor your slip and ask to the regulator date. "If we don't fix this by Friday, we miss OFAC on 14th."

7. Practice 练习题

Exercise 1 — Translate the weekly status

Translate the following weekly status into natural Chinese.

"This week: project RAG is amber. Headline: on track for the MAS TRM milestone on 28th, but M4 SIT slipped by 1 week because the test environment was unavailable for 4 days. Last week we closed M3 build. This week: M4 SIT is at 40% complete, 1 week behind. M5 UAT is on plan to start 22nd. Next week: M4 SIT continues, recovery plan is to add a second test team. RAID: one new issue — the offshore QA contractor missed the onboarding deadline; mitigation is to extend their onboarding by 3 days. Asks: Sponsor to approve the additional contractor cost by Wednesday, and Group Cyber to confirm the pen test report by Friday."

"本周：项目 RAG 琥珀色。一句话：28 号 MAS TRM 里程碑按计划，但 M4 SIT 因测试环境 4 天不可用推迟 1 周。上周关闭 M3 构建。本周：M4 SIT 完成 40%，落后 1 周；M5 UAT 计划 22 日启动，无变化。下周：M4 SIT 继续，恢复计划是增加第二支测试团队。RAID：新登记一个问题——离岸 QA 外包错过入职截止；缓解是将入职延长 3 天。Asks：Sponsor 周三前批准额外外包费用，集团 Cyber 周五前确认渗透测试报告。"

Exercise 2 — Build a weekly status from scratch

You are the PM. Build a 60-second weekly status for: project Cross-Border Payments Modernisation, RAG amber, M3 on track (80% complete, ahead by 2 days), M4 slipped 3 days (root cause: sandbox provisioning delay), one new risk (vendor cutover window not confirmed), two asks (Sponsor sign-off on cutover weekend by Wednesday, Group Cyber to confirm pen test by Friday). Use the Last/This/Next rhythm.

This week: M3 is now 90% complete, M4 SIT started but is 3 days behind. Next week: M3 closes, M4 SIT continues, and we run cutover planning with the vendor on Tuesday. RAID: one new risk — vendor hasn't confirmed the cutover window, probability medium, impact high. Mitigation: cutover workshop Tuesday. Asks: Sponsor to sign-off cutover weekend by Wednesday, Group Cyber to confirm pen test by Friday. Helen, please reflect both in the SteerCo pack. Tom, M3 detail?"
