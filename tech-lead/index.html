Welcome, Tech Lead / 技术负责人你好

As a Tech Lead in a foreign bank, you are the technical authority at the squad level. You run standups, drive sprint planning, lead code reviews, set design quality bars, push back on scope, and handle production incidents — all while speaking English with developers, BAs, PMs, architects, and InfoSec across the UK, Hong Kong, mainland China, and India. This section focuses on the spoken English scenarios you actually face — from sprint health updates to telling a senior dev their design is wrong.

面向外企银行 Tech Lead，本节聚焦你日常真正遇到的口语场景，从 Sprint 状态同步到 code review，从 push back 不合理需求到生产事件指挥。每篇文章都包含真实场景、英文对话、中文 解释、关键句型与练习。

What you'll learn / 你将学到

  • ✅ Run TL standups, sprint planning, backlog refinement, and 1:1s with confidence: sprint health is amber / capacity check / trade sprint scope for / how are you doing — really?
  • ✅ Lead code reviews, pair programming, tech spikes, and tech-debt sprints
  • ✅ Write scannable code review summaries, ADR decision emails, and engineering all-hands updates
  • ✅ Push back on unreasonable deadlines, bad quick-fixes, and senior devs' wrong designs — politely but firmly
  • ✅ Command a P1 production incident, run a war-room, and own the post-mortem
  • ✅ Use banking-specific Tech-Lead terminology correctly: KYC, AML, idempotency, sanctions, regulator milestones, HKT / UK / IN timezones

How to Use This Section / 学习建议

💡 Daily Practice Tip

Pick one article per day. Tech Leads live in meetings — focus on the trade-off framing: always present options, the chosen one, and the consequences. That rhythm is what makes a TL sound senior.

⚠️ For Real Foreign-Bank Work

Always validate architectural and incident decisions with your local Chief Architect, InfoSec, and Risk before circulating widely. Group Standards (SDL, Group Cyber, Group Resilience) override local preferences. Regulator milestones (HKMA, MAS, FCA) are sacred — pull stories, not deadlines.
