Cloud Strategy Direction — Build vs Buy, Multi-Cloud, Regulator Expectation, Exit Cost

Article 07 · Module 2: Strategic Decisions · SM English

⏱ 8 min read 🔴 Expert 🎯 Cloud strategy, build vs buy, multi-cloud, regulator expectation, exit cost

1. Scenario 场景背景

It is Monday 09:30 HKT. You are the Group CIO. The Group Technology Committee meets this afternoon to set the cloud direction for the next 3 years. The current state: 38% of workloads run on cloud (mostly AWS, some Azure), 62% still on on-prem (3 data centres, 22-year-old mainframe, 4 mid-range servers). The 5 cloud provider proposals have been evaluated by Group Architecture and Group Cyber. The HKMA has published its final Cloud Computing Guidance last month. The committee must decide: single-cloud, multi-cloud, or hybrid — and the exit cost envelope. Present: Group CTO, Group CRO, Group CFO, Head of Group Architecture, Head of Group Cyber.

周一 09:30 HKT，你是集团 CIO。集团技术委员会今天下午开会，确定未来 3 年云方向。当前状态：38% 工作负载在云上（主要是 AWS，部分 Azure），62% 仍在本地（3 个数据中心、22 岁主机、4 个中型服务器）。集团架构和集团网络已评估 5 家云厂商提案。HKMA 上月发布最终版云计算指引。委员会必须决策：单云、多云还是混合——以及退出成本额度。参会：集团 CTO、集团 CRO、集团 CFO、集团架构负责人、集团网络负责人。

Cloud strategy is the 5-year bet of any CIO. The pattern: regulator expectation first, build vs buy second, multi-cloud third, exit cost fourth, recommendation last. The regulator now treats cloud concentration as a Tier-2 risk.

云战略是任何 CIO 的5 年赌注。模式：监管期望第一、自建 vs 购买第二、多云第三、退出成本第四、推荐最后。监管现在把云集中度视为 Tier-2 风险。

2. The Cloud Direction — 5 Sections, 60 Minutes

You (CIO): Regulator expectation. HKMA final guidance: 4 explicit asks. Ask 1: data localisation for customer PII — must reside in HK or Group-approved jurisdiction. Ask 2: concentration risk register — top-3 cloud providers disclosed, exit plan documented. Ask 3: sovereign cloud option for regulated workloads. Ask 4: annual attestation to HKMA on cloud resilience.监管期望。HKMA 最终指引：4 项明确要求。要求 1：客户 PII 数据本地化——必须位于香港或集团批准法域。要求 2：集中度风险登记——披露前 3 大云厂商，文档化退出计划。要求 3：受监管工作负载的 sovereign cloud 选项。要求 4：每年向 HKMA 提交云韧性合规声明。
You (CIO): Build vs buy. Build (private cloud on-prem) is rejected — 5-year TCO $620M, regulator expects cloud, talent is hard to retain. Buy (public cloud) is recommended — 5-year TCO $310M, native regulator controls, faster delivery.自建 vs 购买。自建（本地私有云）已否决——5 年 TCO 6.2 亿，监管期望云上、人才难留。购买（公有云）推荐——5 年 TCO 3.1 亿，原生监管控制，交付更快。
Group CRO: CRO: Multi-cloud or single-cloud?CRO：多云还是单云？
You (CIO): Two-cloud strategy. AWS as primary for customer-facing workloads, Azure as secondary for regulated data and DR. Single-cloud rejected: HKMA treats concentration as Tier-2 risk. Three-cloud rejected: operational complexity, no incremental regulator benefit, +18% TCO.双云策略。AWS 作为面向客户工作负载的主云，Azure 作为受监管数据和 DR 的副云。单云否决：HKMA 将集中度视为 Tier-2 风险。三云否决：运营复杂度高，无增量监管收益，TCO 增加 18%。
You (CIO): Exit cost envelope. Exit cost from AWS or Azure: data egress $14M, application re-architecture $42M, contract termination $8M. Total exit cost from one provider: $64M over 18 months. Exit cost is the reason for two-cloud, not feature comparison.退出成本额度。退出 AWS 或 Azure：数据出口 1,400 万、应用重新架构 4,200 万、合同终止 800 万。从一家厂商退出总成本：18 个月内 6,400 万。退出成本是双云的原因，不是功能比较。
Group CFO: CFO: 3-year capex for cloud migration?CFO：3 年云迁移 capex？
You (CIO): 3-year cloud migration capex: $185M. Year 1: $58M (regulator-priority workloads, 18% of estate), Year 2: $72M (core banking partial, 30% of estate), Year 3: $55M (long tail, 52% of estate). Cumulative cloud: 78% by Year 3, mainframe fully decommissioned. Data centres: 3 to 1. Annual run-rate saving at Year 3: $42M.3 年云迁移 capex：1.85 亿。第 1 年：5,800 万（监管优先工作负载，资产 18%），第 2 年：7,200 万（核心银行部分，资产 30%），第 3 年：5,500 万（长尾，资产 52%）。第 3 年累计云上 78%，主机完全下线。数据中心：3 个变 1 个。第 3 年年化运行节省：4,200 万。
You (CIO): Recommendation: two-cloud strategy. AWS primary, Azure secondary. Exit cost envelope $64M per provider, documented in the concentration risk register. 3-year migration capex $185M, inside the $280M envelope. Annual run-rate saving $42M from Year 3. HKMA attestation cycle aligned with annual cloud resilience review. One decision: approve two-cloud direction, $185M 3-year capex.推荐：双云策略。AWS 主、Azure 副。退出成本额度每家 6,400 万，在集中度风险登记中文档化。3 年迁移 capex 1.85 亿，在 2.8 亿额度内。第 3 年起年化节省 4,200 万。HKMA 合规声明周期与年度云韧性评审对齐。一个决策：批准双云方向、3 年 capex 1.85 亿。
CTO: CTO: Two-cloud approved. Exit plan and concentration risk register to HKMA by Q1 next year.CTO：双云批准。退出计划和集中度风险登记明年 Q1 提交 HKMA。

3. Key Phrases 关键句型

  • Regulator expectation, 4 explicit asks HKMA final guidance: 4 explicit asks. Data localisation, concentration risk, sovereign option, annual attestation. HKMA 最终指引：4 项明确要求 "Explicit asks" is the precise phrase. The regulator publishes "asks", not "suggestions". The committee cannot debate what the regulator has asked for.
  • Build vs buy, 5-year TCO Build: 5-year TCO $620M, rejected. Buy: 5-year TCO $310M, recommended. 自建 5 年 TCO 6.2 亿否决；购买 5 年 TCO 3.1 亿推荐 Pair the decision with a TCO. "Build vs buy" without a number is opinion. The CFO approves TCO, not direction.
  • Two-cloud, exit cost is the reason Exit cost is the reason for two-cloud, not feature comparison. 退出成本是双云的原因，不是功能比较 Multi-cloud is not about features. It is about exit cost and concentration risk. The CRO owns the concentration risk, not the feature debate.
  • Exit cost envelope, $64M per provider Total exit cost from one provider: $64M over 18 months. 从一家厂商退出总成本：18 个月内 6,400 万 The exit cost envelope is what makes two-cloud possible. Without an exit cost, the concentration risk is unbounded.
  • Concentration risk register Top-3 cloud providers disclosed, exit plan documented in the concentration risk register. 披露前 3 大云厂商，退出计划在集中度风险登记中文档化 The concentration risk register is a regulator artefact. It is the document the HKMA will read at the next supervisory meeting.
  • Annual run-rate saving Annual run-rate saving at Year 3: $42M. 第 3 年年化运行节省 4,200 万 "Run-rate saving" is the language of CFO. It is the year-3 annual figure, not the 3-year cumulative. Always name the year and the run-rate.

4. Vocabulary 词汇表

  • Single-cloud n. 单云 Using one cloud provider for all workloads.
  • Multi-cloud n. 多云 Using multiple cloud providers (two, three, etc.).
  • Two-cloud strategy n. 双云策略 Using exactly two cloud providers — primary and secondary.
  • Hybrid cloud n. 混合云 A mix of on-prem and public cloud.
  • Sovereign cloud n. 主权云 A cloud deployment in a regulator-approved jurisdiction.
  • Data localisation n. 数据本地化 The requirement that data resides in a specific jurisdiction.
  • Concentration risk n. 集中度风险 The risk that too much workload depends on one provider.
  • Exit cost n. 退出成本 The cost to move away from a cloud provider.
  • Data egress n. 数据出口 The cost of moving data out of a cloud provider.
  • Run-rate saving n. 年化节省 The annual saving once migration is complete.

5. Grammar Notes 语法点

① "Build vs buy" — the binary decision frame

Cloud strategy is presented as a binary: build vs buy, single vs multi, primary vs secondary. The binary forces the committee to choose. Three-way frames ("single, two, or three") let the committee defer. Always end the frame at two.

云战略以二元呈现：build vs buy、single vs multi。二元强迫委员会选择。三元让委员会拖延。

② "Exit cost is the reason, not feature comparison" — the precise rebuttal

When challenged on multi-cloud, the rebuttal is: exit cost is the reason, not feature comparison. The phrase signals the decision is about risk, not preference. The CRO and CFO understand risk; they do not care about feature wars.

当被挑战多云决策时，反驳是：退出成本是原因，不是功能比较。这句话表明决策是关于风险，不是偏好。

6. Cultural Tips 文化 / 职场 Tips

  • Lead with regulator expectation. The committee that ignores the regulator loses. Cloud decisions are regulator decisions now.
  • Build vs buy is a TCO decision. "Cloud-native is the future" is opinion. "Build $620M vs buy $310M, 5-year TCO" is a decision.
  • Multi-cloud is exit cost, not features. The moment you say "AWS has better Lambda", you are in a feature war. The exit cost frame keeps you in risk language.
  • Document the exit cost in the concentration risk register. A number that is not documented is a number that does not exist. The regulator reads the register.
  • Run-rate saving is the year-3 annual figure, not cumulative. "$42M over 3 years" loses. "$42M annual run-rate from Year 3" wins. The CFO remembers the run-rate.
  • Annual attestation is the regulator deadline. The cloud strategy must align with the regulator's annual attestation cycle. The committee approves the cycle, not the strategy alone.

7. Practice 练习题

Exercise 1 — Translate the cloud direction

Translate the recommendation sentence.

"Recommendation: two-cloud strategy. AWS primary, Azure secondary. Exit cost envelope $64M per provider, documented in the concentration risk register. 3-year migration capex $185M, inside the $280M envelope. Annual run-rate saving $42M from Year 3. HKMA attestation cycle aligned with annual cloud resilience review. One decision: approve two-cloud direction, $185M 3-year capex."

"推荐：双云策略。AWS 主、Azure 副。退出成本额度每家 6,400 万，在集中度风险登记中文档化。3 年迁移 capex 1.85 亿，在 2.8 亿额度内。第 3 年起年化节省 4,200 万。HKMA 合规声明周期与年度云韧性评审对齐。一个决策：批准双云方向、3 年 capex 1.85 亿。"

Exercise 2 — Spot the cloud strategy mistake

Which line would lose credibility in a cloud strategy committee? Why?

  1. "AWS has better Lambda, so we should go single-cloud."
  2. "Build: 5-year TCO $620M, rejected. Buy: 5-year TCO $310M, recommended."
  3. "Exit cost envelope $64M per provider."
  4. "Annual run-rate saving $42M from Year 3."

① is the credibility-damager. "AWS has better Lambda" is a feature war, not a strategy. ② pairs build / buy with TCO. ③ names the exit cost. ④ names the run-rate saving. Cloud strategy is TCO + exit cost, not Lambda.

Exercise 3 — Run a 5-minute cloud recommendation

You are the CIO. Build TCO $580M, buy TCO $340M. AWS and Azure both pre-approved by Group Cyber. HKMA requires sovereign option. Recommendation?

"Regulator expectation: HKMA sovereign option mandatory. Build vs buy: build $580M 5-year TCO, rejected — regulator expects cloud. Buy $340M, recommended. Single vs multi: single-cloud rejected — concentration is Tier-2 risk. Two-cloud: AWS primary for customer-facing, Azure secondary for regulated data and sovereign option. Exit cost: $52M per provider over 18 months. 3-year migration capex $165M, inside the $280M envelope. Annual run-rate saving $38M from Year 3. Recommendation: two-cloud, $165M 3-year capex, exit cost documented in concentration risk register."
